# Release Note - News Scraper (v1.2.0)

본 릴리즈는 미국 경제 및 증시 뉴스의 '마감시황(Macro & Market)' 수집 오류(Investopedia 기사 누락, Yahoo Finance 본문 파싱 실패)를 완전 해결하고, 병렬 복호화 및 브라우저 기반 동적 본문 추출 파이프라인을 구축하여 수집 속도와 품질을 비약적으로 끌어올린 고도화 패치입니다.

---

## [v1.2.0] - 2026-06-04

### 📢 주요 변경 사항 (Key Highlights)

1. **Investopedia.com 기사 누락 문제 해결 (도메인 타겟 검색 분할)**
   - 기존의 단일 대용량 검색(max_results=100)으로 기사가 누락되던 구조에서 탈피하여, 각 도메인 전용 쿼리(`site:finance.yahoo.com`, `site:investopedia.com`)로 검색을 분할하여 실행하도록 개선하였습니다.
   - 이를 통해 인베스토피디아의 시황 기사가 항상 최신으로 수집되도록 보장합니다.

2. **고성능 병렬 URL 디코더 구현 (`batch_decode_google_urls`)**
   - 구글 API 변경으로 빈 리스트를 뱉던 기존 라이브러리 디코더 버그를 해결하고, `ThreadPoolExecutor`와 `new_decoderv1`을 결합한 15개 스레드 기반 병렬 복호화 모듈을 신규 탑재하였습니다.
   - 100개 기준 약 16초대의 고속 디코딩을 지원하며, 순차 요청 시 발생하던 누적 차단(HTTP 429) 문제를 근본적으로 예방합니다.

3. **Playwright 동적 본문 추출 및 다중 폴백 시스템 연동**
   - 일반 파서로 수집하기 어렵던 야후 파이낸스 및 인베스토피디아 본문을 수집하기 위해, 헤드리스 Chromium 브라우저(Playwright)를 기동하는 `get_article_text_playwright` 기능을 연동하였습니다.
   - 본문 추출 실패 시 검색 엔진 스니펫 및 제목 유사도 비교를 활용하는 견고한 다중 폴백(Fallback) 요약 체계를 구현하였습니다.

4. **미국 특징주 개별 디코딩 폴백 강화**
   - 미국 특정 기업 특징주 수집(`fetch_company_news_us`) 시에도 개별 복호화 처리를 적용하여 링크가 깨지거나 누락되는 오류를 방지했습니다.

---

### 🛠️ 세부 변경 및 버그 수정 내역

- **Fix**: 야후 파이낸스 및 인베스토피디아의 기사 제목 변동성을 고려하여, 과도하게 엄격했던 키워드 필터링(Regex) 조건을 완화하여 정상 시황 기사가 누락되지 않도록 조치했습니다.
- **QA**: `tests/test_macro_market_scraper.py` 단위 테스트 코드를 신규 작성하여, 각 도메인 기사의 성공적 수집 및 본문 텍스트의 정상 획득 여부를 자동 검증(100% Pass)하였습니다.

## [v1.1.0] - 2026-05-27

### 📢 주요 변경 사항 (Key Highlights)

1. **로컬 구글 뉴스 디코딩 체계 전환 (`googlenewsdecoder` 도입)**
   - 기존의 빈번한 웹 요청 방식의 구글 뉴스 URL 디코딩 로직을 로컬 라이브러리(`googlenewsdecoder`)를 이용한 오프라인 복호화 체계로 100% 전환했습니다.
   - 이를 통해 봇 감지 차단을 완벽히 우회하고 수집 처리 속도를 대폭 단축했습니다.

2. **네이버 모바일 뉴스 검색 스크래핑 체계 구축**
   - 봇 감지 정책이 까다로운 PC 네이버 검색 대신 모바일 검색(`m.search.naver.com`) 및 모바일 헤더를 적용하여 원천적으로 차단을 우회했습니다.
   - 네이버의 잦은 HTML CSS 클래스명 변경(난독화)에 영향받지 않도록, 동적 클래스를 완전 배제하고 **[텍스트-URL 견고한 매칭(Robust Filtering) 알고리즘]**을 도입해 5개의 특징주 기사를 항상 정확하게 수집하도록 개선했습니다.

3. **특징주 뉴스 원본 링크(Hyperlink) 렌더링 지원**
   - 한국 거래대금 상위 100개 종목 테이블의 하단에 출력되는 특징주 상승 이유(뉴스 리스트)를 단순 텍스트가 아닌, 제목과 원본 링크가 결합된 클릭 가능한 마크다운 링크 포맷(`[기사 제목](URL)`)으로 정렬하여 사용자 편의성을 높였습니다.

4. **네이버 OpenAPI 의존성 및 요약 오버헤드 완벽 제거**
   - 뉴스 수집 오버헤드를 유발하고 API 할당량 제한을 겪던 네이버 OpenAPI 요약 hooks를 삭제하고, RSS 본문 추출 및 다중 폴백 요약 알고리즘을 최적화했습니다.

5. **매크로 뉴스 '마감시황' 단독 섹션 렌더링 지원**
   - 연합인포맥스 RSS 등을 통해 수집한 매크로 기사들을 카테고리 기사 목록과 섞지 않고, 보고서의 상단 요약 근처 영역에 **`## 📰 마감시황`**이라는 독자적인 2단계(H2) 마크다운 헤더 아래에 별도로 출력해 가독성을 끌어올렸습니다.

### 🛠️ 세부 변경 및 버그 수정 내역

- **Fix**: `scraper_kr.py` 내부에서 BeautifulSoup의 로컬 섀도우 임포트로 인해 간헐적으로 발생하던 Python `UnboundLocalError` 스코핑 버그를 완전 해결했습니다.
- **Added**: `config/settings.py`에 확장 및 유지보수가 용이한 국내 매크로 뉴스 RSS 피드(`macro_rss_feeds`) 설정 구조를 추가하여 향후 새로운 채널(예: 연합뉴스 등)을 설정만으로 확장할 수 있게 했습니다.
- **QA**: `tests/test_kr_news_changes.py` 테스트 슈트를 작성 및 수행하여 로컬 디코더, 연합인포맥스 RSS, 네이버 모바일 수집기 등의 안정성과 무결성을 성공적으로 검증했습니다.
